1.1 数据传输方式

接口使用Http协议的GET方式传输数据，输出数据格式都为JSON。

    JSON的值只有六种数据类型：
    Number：整数或浮点数
    String：字符串
    Boolean：true 或 false
    Array：数组包含在方括号[]中
    Object：对象包含在大括号{}中
    Null：空类型

1.2 统一格式

接口中所有的时间属性格式统一为：YYYY-MM-DD HH:MM:SS。
接口中所有用到的中文字符串都使用UTF-8编码。
接口中所有的key值都必须是小写字母。
接口中所有的时间戳精确到秒数。

1.6 浏览器禁止cookie解决方案

http无状态，当浏览器禁用cookie后，无法正常使用账户体系。
目前解决方案：在url中加入参数token=3c05fd055fb71ea575495a6224df95e8&uid=14630340981
1.第一次打开页面，用户端检测到没有用户信息。
2.跳转到第三方登录页面
3.服务器获取到用户信息。并且组装带有token、uid的url
4.客户端拿到token、uid，保存到本地（每一个访问的url都带上token、uid）
5.当浏览器没有禁用cookie时，使用保存在cookie中的token、uid和服务器交互
6.当禁止了cookie，由url带参数tokne、uid和服务器交互。

注意组装url时第一个参数为token、第二个参数为uid
